Setup Agent

Overview

This guide covers complete agent deployment for distributed hashcat processing. Agents can run on GPU-enabled servers, local machines, or cloud instances to provide computational power for password cracking jobs.

Prerequisites

System Requirements:
  • Linux/Windows/macOS
  • Go 1.21+ or agent binary
  • Hashcat 6.1.1+
  • Network access to server
Recommended:
  • GPU with CUDA/OpenCL
  • 8GB+ RAM
  • SSD storage
  • Stable internet connection

Quick Setup (5 minutes)

# 1. Download agent binary curl -L https://github.com/your-repo/releases/latest/download/agent-linux -o agent chmod +x agent # 2. Create directories mkdir -p /root/uploads/{wordlists,hash-files,temp} # 3. Start agent ./agent --server http://your-server:1337 --name my-gpu-agent

That's it! Your agent is now connected and ready to process jobs.

Manual Installation

1Install Dependencies
Ubuntu/Debian:
# Update system sudo apt update && sudo apt upgrade -y # Install dependencies sudo apt install -y curl git build-essential # Install hashcat sudo apt install -y hashcat # Install NVIDIA drivers (if GPU) sudo ubuntu-drivers autoinstall sudo reboot
CentOS/RHEL:
# Update system sudo yum update -y # Install dependencies sudo yum install -y curl git gcc # Install hashcat from EPEL sudo yum install -y epel-release sudo yum install -y hashcat # Install NVIDIA drivers sudo yum install -y nvidia-driver
Windows:
# 1. Download hashcat from: # https://hashcat.net/hashcat/ # 2. Install NVIDIA/AMD drivers # 3. Download agent.exe from releases # 4. Run in PowerShell: .\agent.exe --server http://server:1337
2Setup Directories
# Create upload structure sudo mkdir -p /root/uploads/{wordlists,hash-files,temp} sudo mkdir -p /root/uploads/wordlists/{common,leaked,custom} sudo mkdir -p /root/uploads/hash-files/{wifi,other} # Set permissions sudo chown -R $USER:$USER /root/uploads sudo chmod -R 755 /root/uploads # Verify structure tree /root/uploads
3Download & Build Agent
Option A: Pre-built Binary
# Download latest release curl -L https://github.com/your-repo/go-distributed-hashcat/releases/latest/download/agent-linux \ -o /usr/local/bin/hashcat-agent # Make executable sudo chmod +x /usr/local/bin/hashcat-agent # Test installation hashcat-agent --version
Option B: Build from Source
# Install Go 1.21+ wget https://go.dev/dl/go1.21.0.linux-amd64.tar.gz sudo tar -C /usr/local -xzf go*.tar.gz export PATH=$PATH:/usr/local/go/bin # Clone and build git clone https://github.com/your-repo/go-distributed-hashcat.git cd go-distributed-hashcat make build-agent # Install binary sudo cp bin/agent /usr/local/bin/hashcat-agent
4Configure Agent
Create Configuration File:
# Create config directory sudo mkdir -p /etc/hashcat-agent # Create configuration file sudo tee /etc/hashcat-agent/config.yaml << 'EOF' server: url: "http://your-server:1337" agent: name: "gpu-server-01" upload_dir: "/root/uploads" max_jobs: 4 heartbeat_interval: "30s" gpu: enabled: true vendor: "nvidia" # or "amd" memory: "24GB" compute_capability: "8.6" logging: level: "info" file: "/var/log/hashcat-agent.log" EOF
Alternative: Environment Variables
# Set environment variables export HASHCAT_SERVER_URL="http://your-server:1337" export HASHCAT_AGENT_NAME="gpu-server-01" export HASHCAT_UPLOAD_DIR="/root/uploads" export HASHCAT_MAX_JOBS="4" export HASHCAT_GPU_INFO="RTX 4090, 24GB VRAM"
5Setup as System Service
Systemd Service (Linux):
# Create systemd service sudo tee /etc/systemd/system/hashcat-agent.service << 'EOF' [Unit] Description=Hashcat Distributed Agent After=network.target [Service] Type=simple User=root WorkingDirectory=/opt/hashcat-agent ExecStart=/usr/local/bin/hashcat-agent \ --server http://your-server:1337 \ --name gpu-server-01 \ --upload-dir /root/uploads \ --max-jobs 4 Restart=always RestartSec=10 Environment=HASHCAT_WORKLOAD_PROFILE=3 Environment=CUDA_VISIBLE_DEVICES=0,1 StandardOutput=journal StandardError=journal [Install] WantedBy=multi-user.target EOF # Enable and start service sudo systemctl daemon-reload sudo systemctl enable hashcat-agent sudo systemctl start hashcat-agent # Check status sudo systemctl status hashcat-agent
Docker Service:
# Create docker-compose.yml cat > docker-compose.yml << 'EOF' version: '3.8' services: hashcat-agent: image: hashcat-agent:latest container_name: hashcat-agent restart: always runtime: nvidia environment: - NVIDIA_VISIBLE_DEVICES=all - HASHCAT_SERVER_URL=http://your-server:1337 - HASHCAT_AGENT_NAME=docker-gpu-agent volumes: - ./uploads:/root/uploads - /dev:/dev network_mode: host privileged: true EOF # Run with Docker Compose docker-compose up -d # Check logs docker-compose logs -f hashcat-agent

Agent Commands & Options

Basic Usage:
# Start agent with minimum options ./agent --server http://localhost:1337 # Start with custom name and capabilities ./agent --server http://192.168.1.100:1337 \ --name "my-gpu-rig" \ --capabilities "RTX 4090, 24GB VRAM" # Start with custom upload directory ./agent --server http://server:1337 \ --upload-dir /custom/path \ --name "dedicated-gpu"
Advanced Options:
# Full configuration ./agent \ --server http://server:1337 \ --name "production-gpu-01" \ --ip 192.168.1.50 \ --port 8080 \ --capabilities "RTX 4090, CUDA 12.0" \ --upload-dir /data/uploads \ --max-jobs 8 \ --heartbeat-interval 15s \ --log-level debug \ --config /etc/agent/config.yaml
Available Command Line Options:
--server - Server URL (required)
--name - Agent name
--ip - Agent IP address
--port - Agent port (default: 8081)
--capabilities - Hardware description
--upload-dir - Local files directory
--max-jobs - Concurrent jobs limit
--heartbeat-interval - Heartbeat frequency
--log-level - Logging level

Monitoring & Troubleshooting

Monitor Agent Status:
# Check service status sudo systemctl status hashcat-agent # View logs sudo journalctl -u hashcat-agent -f # Check agent registration curl http://server:1337/api/v1/agents/ # Monitor GPU usage nvidia-smi -l 1
Common Issues:
Agent not registering:
Check network connectivity and server URL
GPU not detected:
Install proper drivers and verify with nvidia-smi
Jobs failing:
Check hashcat installation and file permissions
Performance Optimization:
# Set GPU performance mode sudo nvidia-smi -pm 1 # Set power limit (adjust for your GPU) sudo nvidia-smi -pl 300 # Enable persistence mode sudo nvidia-smi -pm 1 # Check GPU temperature nvidia-smi --query-gpu=temperature.gpu --format=csv,noheader

Security Considerations

Network Security:
  • • Use VPN or private networks when possible
  • • Restrict agent access to necessary ports only
  • • Use firewall rules to limit connections
  • • Consider using reverse proxy with authentication
System Security:
  • • Run agents with limited user privileges
  • • Regularly update system and dependencies
  • • Monitor agent logs for suspicious activity
  • • Use disk encryption for sensitive data

Performance Metrics Explanation:
  • • Hit Rate: Percentage of requests served from cache
  • • Query Reduction: Estimated database query reduction (90% of hit rate)
  • • Response Speed: Estimated response speed improvement (95% of hit rate)
  • • Cache Entries: Total number of cached objects (agents + wordlists + hashFiles)
