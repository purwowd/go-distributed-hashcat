Quick Start Guide

Authentication

Current Version: No Authentication Required

The current API version does not require authentication. All endpoints are publicly accessible. Future versions will include API key authentication for enhanced security.

Planned Authentication (Future)

API Key Authentication

Future versions will require API key in headers:

curl -H "X-API-Key: your-api-key" \
  http://localhost:1337/api/v1/agents
Agent Key Authentication

Agents will authenticate using pre-shared keys:

curl -H "Authorization: Bearer agent-key" \
  http://localhost:1337/api/v1/agents/register

Complete WiFi WPA/WPA2 Cracking Workflow

Step-by-step guide to crack WiFi passwords using distributed hashcat

1

Upload WiFi handshake file

Upload your captured WPA/WPA2 handshake file (.hccapx or .22000 format)

curl -X POST -F "file=@wifi_handshake.hccapx" \
  http://localhost:1337/api/v1/hashfiles/upload
Response: {"id": "hash-file-uuid", "filename": "wifi_handshake.hccapx"}
2

Upload wordlist

Upload your wordlist for dictionary attack (e.g., rockyou.txt)

curl -X POST -F "file=@rockyou.txt" \
  http://localhost:1337/api/v1/wordlists/upload
Response: {"filename": "rockyou.txt", "size": "139MB"}
3

Create hashcat job

Create a new job linking the hash file and wordlist

curl -X POST -H "Content-Type: application/json" \
  -d '{
    "name": "WiFi Network Crack",
    "hash_file_id": "hash-file-uuid",
    "wordlist": "rockyou.txt",
    "attack_mode": 0,
    "hash_type": 2500
  }' \
  http://localhost:1337/api/v1/jobs
Response: {"id": "job-uuid", "status": "created"}
4

Start the job

Begin distributed cracking across all available agents

curl -X POST http://localhost:1337/api/v1/jobs/job-uuid/start
5

Monitor progress

Check job status and progress in real-time

curl http://localhost:1337/api/v1/jobs/job-uuid
Response includes: progress percentage, speed, ETA, and found passwords

Real-time Updates with WebSocket

Connect to WebSocket for live job progress and system notifications

Connection

WS ws://localhost:1337/ws
const ws = new WebSocket('ws://localhost:1337/ws')

ws.onopen = () => {
  console.log('Connected to hashcat WebSocket')
}

ws.onmessage = (event) => {
  const data = JSON.parse(event.data)
  console.log('Received update:', data)
  
  // Handle different event types
  switch(data.type) {
    case 'job_progress':
      updateProgressBar(data.progress)
      break
    case 'password_found':
      displayFoundPassword(data.password)
      break
    case 'agent_status':
      updateAgentStatus(data.agent_id, data.status)
      break
  }
}

ws.onclose = () => {
  console.log('WebSocket connection closed')
}

ws.onerror = (error) => {
  console.error('WebSocket error:', error)
}

Real-time Event Types:

job_progress Job completion percentage
password_found Cracked password discovered
job_complete Job finished successfully
agent_status Agent online/offline status
job_error Job execution errors
system_notification General system messages
